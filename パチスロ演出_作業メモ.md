## トリガー条件（SlotTrigger.check の判定順）

1. 記録が **777 kcal** → フィーバー（激アツ）
2. 今日の合計が **目標カロリーぴったり** → ぴったり賞（激アツ）
3. 記録カロリーが **ゾロ目**（111以上: 111, 222, …, 999, 1111…）→ 当たり
4. 今日の合計がゾロ目 → 当たり
5. **夕食を記録して合計が目標内**（目標の50%以上）→ 目標達成の当たり

## 通知の理路整然化（2026-07-02 追加）

旧実装の問題: メッセージがスケジュール時に1回だけ選ばれ毎日同じ内容が繰り返される＋
「もう既にカロリーオーバー」等、実際の記録状態と無関係な断定メッセージが飛んでいた。

新設計（`lib/services/notification_service.dart` 全面書き換え）:

- `NotificationService.reschedule(storage)` に一本化。**起動時・食事の記録/削除時・目標変更時・通知オン時**に全通知を組み直す
- 今日の残り枠（8/12/19時）は組み直し時点の実データで文面生成:
  - 未記録 → 未記録の指摘（記録すれば即組み直されるので実態とズレない）
  - 目標オーバー → 実数値入りの注意（「現時点で1900カロリーぽん。目標の1800をもうオーバーぽん」）
  - 目標内 → 実数値入りの進捗（「ここまで800カロリー、残り1000ぽん」）
  - 朝8時は未記録でも煽らない（朝に未記録は普通なので）
- 明日以降6日分は状態を断定しない汎用メッセージを日替わりローテーション（各枠7〜8種に増量）
- 7日目の枠だけ毎日リピート（`matchDateTimeComponents: time`）にして、アプリを1週間開かなくても通知が途切れないフォールバック
- 通知ID: `dayOffset * 100 + hour`
- フック追加: `main.dart`（起動時）、`food_log_screen.dart`（保存・スワイプ削除）、`settings_screen.dart`（通知トグル・目標変更）

## スキャン推定値の一桁ゆらぎ（2026-07-02 追加）

AIの推定カロリーはほぼ必ず10・50の倍数で返るため、スロットのゾロ目・777が
スキャン経由では事実上出ない問題があった（合計ゾロ目は10の倍数の合計だと数学的に不可能）。
→ `openai_service.dart` の `_dejitterRound()` で、5の倍数かつ30以上の推定値に±7のゆらぎを付与。
概算精度は変わらず、見た目の説得力も上がり、ゾロ目が現実的な確率（3桁で約1%）で出るようになった。
手動入力はそのまま（ユーザーが入れた数字を尊重）。

## ドット絵ミニたぬき（2026-07-02 プロトタイプ）

画面下端をうろちょろするドット絵ぽんぽこ。`lib/widgets/mini_tanuki.dart`（新規）＋ main.dartのbodyをStack化して全画面に被せ。

- スプライトは**画像ファイルなし**: 16x16のピクセルマップ（文字列リスト）をCustomPainterで直接描画
- うろちょろAI: 待機(1.5〜5.5s) → ランダムな目的地へ歩行(36px/s) → 待機… の繰り返し。進行方向で左右反転
- 歩行中はワドル（±0.09radの揺れ＋上下バウンス）、2〜5秒ごとにまばたき（目ピクセルをマスク色で塗る）
- タップで軽ハプティクス＋ジャンプ＋吹き出しセリフ（「見てないで記録するぽん」等5種）
- たぬき本体以外はタッチ素通し（Stackのヒットテスト仕様を利用）
- 拡張候補: カロリー状態で歩き方変化、深夜は隅で寝る、ストリーク🔥エフェクト、金たぬき
- **次回の宿題（ユーザーフィードバック）**: 「動いてるけど手足がない四角いたぬき」→ ドット絵の精度を上げる。
  - キャンバスを16x16 → 24x24〜32x32に拡張して手足・しっぽ・輪郭の丸みを表現
  - 歩きは2〜4コマの足パタパタアニメに（今はワドル回転でごまかしている）
  - 横向き歩行スプライト（しっぽが見える）も検討。しっぽの縞模様はたぬきらしさの要
  - Pythonプレビュー（スクラッチパッドのtanuki_v3.png参照）で見た目を詰めてからDartに移植する手順が有効だった

## 効果音（2026-07-04 実装済み、未コミット）

Pythonで8bitレトロ風のWAVを自作合成（フリー素材なし、ライセンス完全フリー）。

- 生成スクリプト: **`tools/gen_sfx.py`**（標準ライブラリのみ、numpy不要）。
  矩形波/三角波/ノイズ＋エンベロープの自作シンセ。実行すると `assets/sounds/` に7ファイル出力
- 音源（22050Hz/16bit/mono、合計約210KB）:
  `spin_loop`（回転ジー＋カタカタ、tick周期の整数倍でシームレスループ）／
  `reel_stop`（ピコッ）／`reach`（上昇スイープ×2）／`cutin`（ズバーン: ノイズ＋低音ブーム）／
  `fanfare_win`（アルペジオ→和音 0.8s）／`fanfare_jackpot`（タタタ×2→ジャーン 2.0s）／
  `pon`（たぬき「ぽんっ」: 破裂ノイズ＋ピッチ跳ね上げ三角波）
- 再生: `audioplayers ^6.1.0` 追加。`lib/services/sfx_service.dart` 新設。
  単発は使い捨てAudioPlayer（完了時dispose）、回転ループだけ共有プレイヤー。
  **マナーモードに従う**（respectSilence: true）＋他アプリの音楽と共存（mixWithOthers）。
  再生失敗は握りつぶしてアプリを止めない
- 組み込み: `slot_jackpot.dart`（回転開始でループ→停止ごとにreel_stop→リーチ/カットイン→
  結果でループ停止＋rank別ファンファーレ、disposeでもループ停止保険）、
  `mini_tanuki.dart`（タップで pon）
- シミュレータ（iPhone 16 Pro）で777フィーバー一式を実行、例外・アセット読込エラーなしを確認済み。
  **音質の耳チェックとマナーモード挙動は実機/シミュレータでユーザー確認待ち**
